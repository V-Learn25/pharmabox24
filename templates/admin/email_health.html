Reading the results

  • HTTP 200 with verified domains shown = API key works. If sends still fail, the issue is with the From address in MAIL_FROM not matching a verified domain.
  • HTTP 401 = API key wrong, revoked, or whitespace in the env var. Regenerate on Resend → Railway → variables → restart.
  • HTTP 403 on a send to an external address = MAIL_FROM uses the sandbox sender (onboarding@resend.dev) or an unverified domain. Verify your domain on Resend and update MAIL_FROM.
  • Network error = Railway outbound networking issue. Rare; usually resolves on its own.
